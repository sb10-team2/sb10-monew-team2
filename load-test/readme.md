## 🚀 부하 테스트 (Load Testing)

본 프로젝트는 서비스 런칭 전, 트래픽 급증 상황에서도 서버가 안정적으로 동작하는지 검증하기 위해 부하 테스트를 진행했습니다.

### 1. 테스트 도구: k6
부하 테스트 도구로는 **k6**를 채택했습니다.
* **가벼운 스레드 모델**: k6는 Go 언어로 개발되었으며, OS 스레드 대신 가벼운 `goroutine`을 사용하여 가상 유저(VU)를 생성합니다.
* **클라이언트 병목 방지**: 이 덕분에 적은 리소스로도 엄청난 양의 동시 요청을 만들어낼 수 있으며, 부하를 발생시키는 클라이언트 쪽에서 먼저 뻗어버리는 현상(Client Bottleneck) 없이 **백엔드 서버의 순수 성능 한계를 정확히 측정**할 수 있습니다.
* **유연한 스크립팅**: 내장된 JS 런타임을 통해 JavaScript/TypeScript로 복잡한 사용자 시나리오를 쉽게 작성할 수 있습니다.

### 2. 사용자 시나리오 설계 (User Journey)
부하 테스트의 핵심은 **'실제 유저가 서비스를 이용하는 패턴'**을 최대한 유사하게 구현하는 것입니다.
뉴스 기사를 읽고(`모뉴`), 댓글을 작성하는 사용자의 행동 패턴을 반영하기 위해 각 액션 사이에 실제 사람의 생각/읽기 시간(`sleep`)을 부여하여 현실성을 높였습니다.

**📌 시나리오 흐름**
1. **회원가입 및 로그인**
2. **최근 기사 탐색**: 최근 생성된 기사 100개를 순회합니다.
3. **기사 상세 조회**: 매 기사마다 뉴스 소스를 확인하고, 단건 조회 또는 Cursor Pagination을 통한 조회를 수행합니다.
4. **상호작용 (댓글/좋아요)**: 해당 뉴스 기사에 임의의 개수(1~N개)로 댓글을 작성하고, 기존 댓글들에 임의로(1~M개) 좋아요를 남깁니다.
5. **알림 및 관심사**: 알림 목록을 확인한 뒤, 새로운 관심사를 생성하고 확인합니다.

*(💡 참고: 완벽한 전체 데이터 탐색을 구현하기보다는 테스트의 일관성을 위해 최근 100개의 기사로 범위를 한정하고, 댓글 및 좋아요 수를 난수화하여 변수를 통제했습니다.)*

### 3. 테스트 진행 단계 (Phases)
정확한 데이터 측정을 위해 테스트는 크게 3가지 단계로 나뉘어 진행됩니다.

* **Phase 0. 데이터 생성 (Data Generation)**
    * 본격적인 테스트 전, 임의의 더미 유저들을 무수히 생성하여 최근 기사 100개에 마구잡이로 댓글과 좋아요를 생성합니다.
    * 이는 DB에 충분한 볼륨(Data Volume)을 쌓아두어, 실제 부하 테스트 시 **조회 쿼리에 현실적인 부하를 주기 위한 사전 작업**입니다.
* **Phase 1. 웜업 (Warm-up)**
    * 시스템에 서서히 부하를 주며 백엔드 서버의 **JIT(Just-In-Time) 컴파일러 최적화**와 **DB 캐시 히트(Cache Hit)**가 충분히 이루어지도록 돕는 예열 단계입니다.
* **Phase 2. 본 테스트 (Load Test)**
    * 예열이 끝난 후, 목표한 최대 가상 유저(Target VUs)를 투입하여 시스템의 한계치와 병목 지점을 집중적으로 모니터링합니다.

### 4. 주요 성능 지표 (Metrics)
테스트 결과 리포트에서는 다음 지표들을 중점적으로 분석합니다.

* **요청 횟수 (Request Count)**: 테스트 기간 동안 발생한 총 HTTP 요청 수
* **TPS / RPS**: 초당 처리한 트랜잭션/요청 수 (시스템의 처리 용량)
* **Error Rate (%)**: 전체 요청 중 실패(5xx, 4xx 에러 등)한 요청의 비율
* **평균 응답 시간 (Avg Response Time)**: 전체 요청의 평균 처리 시간
* **p95 / p99 응답 시간 (Tail Latency)**:
    * 전체 응답 중 상위 95%, 99% 사용자가 경험한 응답 시간입니다.
    * 예를 들어 **p95가 2.3초**라면, 전체 요청의 **95%는 2.3초 이하로 빠르게 처리**되었음을 의미합니다. 소수의 사용자(나머지 5%)가 겪는 극단적인 지연(지표 튀는 현상)을 파악하는 핵심 지표입니다.
